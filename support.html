4K Capture Support

Support page for capture workflows involving capture cards, game consoles, cameras, webcams, and other HDMI sources on Windows.

Contact

Support email: h3r2015@gmail.com

Recommended information to include in a support request

  • Windows version
  • Capture device model
  • Source type, for example game console, camera, TV box, or webcam
  • Audio device model
  • Whether the issue happens in portable, setup, or MSIX build
  • Steps to reproduce the issue
  • A screenshot or short screen recording if possible

Common checks

  • Reconnect the capture card directly to a stable USB 3.x or PCIe connection
  • Close other apps that may already be using the webcam or capture device
  • If you are using a game console or another HDMI source, make sure the source output matches a mode supported by the capture card
  • If 4K60 is unstable, try 4K30 or 1080p60 first
  • If automatic audio selection is wrong, choose the audio input manually
  • Update GPU drivers and capture device drivers when needed

ADB remote notes

  • The target Android TV or TV box must have network ADB enabled
  • The app can scan the local network for reachable ADB endpoints
  • The bundled `adb.exe` helper is included only for the optional remote feature
